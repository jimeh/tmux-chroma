Behavior

Recognize the machine before reading its name.

Chroma hashes tmux’s short hostname into one of seventeen balanced accents. The result is stable on every reload and distinct across the machines you use most.

Host-seeded
Zero configuration by default. The same host always resolves to the same preset.
Quiet context
Prefix and synchronized-pane state become visible without shifting the centered window list.
Useful metrics
CPU and memory are included. Disk is available when you want it, with native implementations for macOS and Linux.
Configure

Start automatic. Override only what matters.

Set options before TPM initializes. Pin a preset, choose your own accent, enable Powerline dividers, or change the information in either side of the status bar. 

# Optional overrides
set -g @chroma_preset 'ember'
set -g @chroma_powerline 'on'
set -g @chroma_show_disk 'on'
set -g @chroma_disk_path '/'

# Load with TPM
set -g @plugin 'jimeh/tmux-chroma'
run '~/.tmux/plugins/tpm/tpm'
@chroma_preset
Use a named accent instead of the host seed.
@chroma_base_color
Supply any full #rrggbb accent value.
@chroma_powerline
Use Powerline dividers when your font supports them.
@chroma_left_extra / right_extra
Add your own compact status content.
Install

One plugin line, then reload tmux.

TPM is the simplest path. Manual installation works too, and keeps the theme entirely transparent.

With TPM

Add Chroma before TPM’s initialization line, then press prefix + I.

set -g @plugin 'jimeh/tmux-chroma'

Manually

Clone the repository and run the plugin after your own overrides.

git clone https://github.com/jimeh/tmux-chroma.git ~/.tmux/plugins/tmux-chroma run-shell "${HOME}/.tmux/plugins/tmux-chroma/chroma.tmux"

Requirements

Chroma supports macOS and Linux with tmux 3.2 or later. Linux needs Bash 4.2 or later; the Bash included with macOS is supported. Powerline mode also needs a font containing the divider glyphs.
